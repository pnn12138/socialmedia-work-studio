# Visual Reviewer - 视觉审查技能

## 技能描述

视觉审查层核心技能，负责独立审核图文成品的主题贴合度、视觉质量、平台适配度与素材规范，输出明确修改意见与发布结论。

此技能不参与前期策划，也不负责执行制作，专门找问题、拦低质量内容。

## 快速开始

```bash
# 审核图文成品
/visual-reviewer {topic-name} --input topics/{topic-name}/fig/visual_plan.md

# 审核单张图片
/visual-reviewer --image topics/{topic-name}/fig/output/01_cover.png --spec topics/{topic-name}/fig/specs/01_cover.json

# 批量审核
/visual-reviewer {topic-name} --all
```

## 审核对象

默认读取：

1. `topics/{topic-name}/post.md` - 母稿
2. `topics/{topic-name}/fig/visual_plan.md` - 视觉规划与执行全案（核心审核文件）
3. `topics/{topic-name}/fig/output/` - 定稿图目录
4. `topics/{topic-name}/xiaohongshu.md` - 小红书适配稿
5. 其他平台适配稿（如有）

**重要**：审核意见必须填写在 `visual_plan.md` 的「审核意见」章节中，不单独创建文件。

## 审核重点

### 1. 主题贴合度

- 每张图是否真的服务正文
- 是否存在"图很好看但与主题关系弱"
- 封面是否抓住文章真正的冲突点

### 2. 图文一致性

- 图片与文字是否相互强化
- 是否出现图文说的不是一回事
- 是否存在夸张封面但正文支撑不足

### 3. 视觉质量

- 是否有廉价 AI 感
- 是否有素材感、拼贴感、杂乱感
- 是否有明显排版问题
- 是否具有统一性和完成度

### 4. 信息清晰度

- 读者是否能一眼看懂主次关系
- 金句图是否真有传播价值
- 信息图是否过载或过空

### 5. 平台适配度

- 小红书：是否具备点击欲和滑读节奏
- 头条：是否足够信息化
- 公众号：是否适合长文插图阅读

### 6. 素材与规范

- 来源是否记录
- 是否存在版权或 attribution 风险
- 是否存在截图质量问题
- 是否存在低分辨率问题

## 输出结论

必须给出以下三种之一：

- `pass` - 通过定稿
- `revise` - 需要修改
- `reject` - 驳回重做

## 逐项问题说明（如需要修改）

对于审核结论为「🔧 修改」的图，必须在下方补充说明：

```markdown
### 01_cover 修改意见
- **问题**：封面标题文字对比度不足，深色背景上不够清晰
- **严重程度**：medium
- **修改建议**：
  1. 增加标题文字亮度或改用白色
  2. 在文字下方增加半透明深色蒙版
  3. 或增加文字描边/阴影效果
- **责任方**：post-builder
```

## 评分标准

### 星级说明

| 星级 | 说明 |
|------|------|
| ★★★★★ | 优秀 - 超出预期 |
| ★★★★☆ | 良好 - 符合标准 |
| ★★★☆☆ | 合格 - 勉强可用 |
| ★★☆☆☆ | 不合格 - 需修改 |
| ★☆☆☆☆ | 差 - 需重做 |

### 各维度评分标准

**主题贴合度**：
- 5 星：图与主题完美契合，强化核心观点
- 4 星：图与主题相关，服务正文
- 3 星：图与主题有关联但较弱
- 2 星：图与主题关系模糊
- 1 星：图与主题无关或跑题

**视觉质量**：
- 5 星：高级感强，审美在线，无廉价 AI 感
- 4 星：视觉质量良好，无明显问题
- 3 星：视觉质量普通，有小问题
- 2 星：视觉质量较差，有明显问题
- 1 星：视觉质量差，无法使用

**平台适配度**：
- 5 星：完美适配平台，传播感强
- 4 星：适配平台，符合规范
- 3 星：基本适配，有小问题
- 2 星：适配度较差
- 1 星：完全不适配

## 审核流程

```
读取 visual_plan.md 和定稿图
    ↓
逐图审核（主题贴合度、视觉质量、平台适配度、素材规范）
    ↓
在 visual_plan.md 中填写审核意见
    ↓
给出审核结论：pass / revise / reject
    ↓
如为 revise/reject，通知 post-builder 返工
    ↓
返工后重新审核，直到 pass
```

## 审核原则

- 不接受"勉强能发"
- 不接受"图比文弱很多"
- 不接受"只是完成任务"
- 不接受"审美普通但没有明显错误"就直接通过
- 不接受"图文割裂"
- 不接受"空泛科技风替代真实主题"

## 必须避免的问题

以下问题出现任何一项，都应给出 `revise` 或 `reject`：

### 视觉问题
- 廉价 AI 感（过度光效、塑料质感）
- 素材堆砌感（元素杂乱无章）
- 版式拥挤（留白不足、信息过载）
- 颜色脏乱（配色不干净、饱和度过高）
- 字体气质混乱（风格不统一）
- 封面标题与背景打架（对比度不足）
- 无意义光效、粒子、发光边框
- 典型"科技蓝模板图"
- 与正文无关的空泛氛围图
- 过度严肃、冰冷、距离感强的科技风

### 内容问题
- 图文不一致
- 封面夸张但正文支撑不足
- 信息错误或不准确
- 缺少必要的来源说明
- 版权风险未标注

### 平台问题
- 尺寸不符合平台规范
- 文字过小无法清晰阅读
- 不符合平台用户阅读习惯

## 与其他技能协作

| 技能 | 协作内容 |
|------|----------|
| `visual-planner` | 审核视觉方案是否被正确执行 |
| `post-builder` | 审核成品质量，给出修改意见 |
| `image-card-builder` | 审核渲染质量 |
| `asset-sourcer` | 审核素材质量与来源规范 |

## 返工流程

当审核结论为 `revise` 时：

```
visual-reviewer 输出修改意见
    ↓
post-builder 根据意见返工
    ↓
visual-reviewer 重新审核
    ↓
直到 pass 或判定需 reject
```

## 环境要求

- 访问 `topics/` 目录的读写权限
- 能够读取图片文件（用于质量检查）

## 最佳实践

1. **先读母稿** - 理解文章核心观点，才能判断图是否贴题
2. **逐图审核** - 不要笼统评价，每张图都要具体意见
3. **意见具体** - 不要说"不好看"，要说"哪里不好看，怎么改"
4. **标准一致** - 对所有图使用同样的审核标准
5. **记录完整** - 在 visual_plan.md 中完整记录审核意见
6. **跟进返工** - 返工后要重新审核，不要一放了之
