# 执行摘要

本文设计并分析了在 Claude Code 平台上实现“图需求规划（visual-planner）→素材检索（asset-sourcer）→图像合成（image-builder）”端到端能力升级的方案。总体目标是让模型根据用户的图像需求，自动生成设计方案并搜集相关素材，最后合成目标图像。方案覆盖三大模块：**Visual Planner**（解析需求、输出结构化设计规范）、**Asset Sourcer**（调用各种图片/API服务、收集素材）和**Image Builder**（图像处理与合成），并分别明确了其输入输出格式、最简可行功能、推荐工具、所需权限、错误处理和监控方式。资产检索部分列举了常用图片API（Pexels、Unsplash、Pixabay、Getty等）、图像生成/编辑API（OpenAI DALL·E/GPT-Image、Stable Diffusion/Replicate、ClipDrop、Remove.bg等）及浏览器自动化工具（Playwright、Stagehand、Browser Use），并分析了各自优缺点、调用示例、速率限制和版权风险。图像合成部分提出了可组合的处理工具链（如 Python Pillow、ImageMagick、Canvas、HTML/CSS 排版引擎等），给出示例工作流和关键实现点（自动裁剪、文字排版、图层管理、多分辨率导出）。最后制定了阶段性实施计划和一系列具体执行指令，每步包括目标、输入示例、预期输出、调用的技能/API示例及错误回退方案，以指导 Claude Code 平台逐步实现该功能。报告中引用了官方文档和社区资料（如 Pexels API 文档【20†L59-L68】【20†L139-L148】、Unsplash API 文档【28†L118-L127】【28†L235-L244】、Pixabay API 文档【30†L83-L92】【30†L119-L127】、OpenAI 图像API【36†L926-L933】、Clipdrop API【45†L106-L109】、Remove.bg API【47†L71-L74】、Stagehand 框架【51†L330-L339】、Browser Use 文档【13†L440-L448】、Playwright 截图功能【48†L105-L113】等）作为优先参考来源。

## 一、目标与范围

本项目目标是在 Claude Code 环境下，实现从用户图像需求（如社交媒体封面、海报设计、幻灯片插图等）的高层次描述，到具体素材选取，再到最终图像输出的全流程自动化。范围包括三个主要模块：

- **Visual Planner**：接受用户的图像需求（通常为自然语言描述），输出结构化的设计规范（JSON格式），包括画面元素、布局提示、样式或主题要素等。示例：用户输入“设计一个春季促销海报，需要花卉背景和醒目的文字”。输出可能包括需要的图层（“背景图案”、“主文案文字”、“logo位置”等）、关键文字内容、配色建议等。

- **Asset Sourcer**：根据 Visual Planner 输出的规范，自动调用素材服务/API检索并下载图像素材。例如，它可能对“花卉背景”关键词调用 Pexels/Unsplash API获取合适图片，对 logo 则可能从企业素材库下载，或调用图片生成模型生成特定元素。该模块需要处理多源调用，包括免费图库API、付费图库、AI图像生成和浏览器自动化抓取。

- **Image Builder**：将检索到的素材按照规划合成最终图像。包括图像处理（裁剪、缩放、颜色校正）、文字排版（字体选择、字号、位置）、图层管理（背景、主体、文字等层叠）和输出多种分辨率（标准、Retina等）。使用可编程的图像库或引擎（Pillow、ImageMagick、HTML/CSS+截图等）来实现。

模块之间的调用和数据交换通过 JSON 等格式完成。未明确的运行环境、预算、权限均视为“未指定”。在各模块设计中，将详细说明输入/输出格式、核心功能、所需技能与工具、权限要求，以及错误处理和可观察性方案。

## 二、Visual Planner 设计

**1. 输入/输出格式**

- *输入格式*：用户自然语言需求或任务描述。例如：`"创建一个简洁的科技类海报，背景为夜空，主题文字为“未来已来”"`。
- *输出格式*：JSON 结构，示例 schema：

```json
{
  "projectType": "poster",
  "theme": "科技,未来",
  "elements": [
    {
      "role": "background",
      "type": "image",
      "description": "夜空背景，带星点",
      "styles": {"brightness": "低", "hue": "偏蓝"}
    },
    {
      "role": "title",
      "type": "text",
      "content": "未来已来",
      "font": {"size": 48, "color": "白色", "style": "无衬线加粗"},
      "position": "center-top"
    },
    {
      "role": "logo",
      "type": "asset",
      "description": "公司Logo",
      "source": "企业素材库"
    }
  ],
  "layoutHints": {
    "alignment": "居中",
    "padding": {"top": 50, "left": 40}
  }
}
```

*说明*：上例中 `"elements"` 列表逐项定义了所需元素：背景图像、标题文字、Logo 等，每项包含类型（图像/文字/素材）、描述和样式提示。`"layoutHints"` 给出整体布局提示。实际 MVP 可简化为识别元素与关键词即可。

**2. 最小可行产品（MVP）功能清单**

- 解析用户描述中的关键信息（主题、对象、文案、风格）。
- 识别所需元素类型（背景图、主体图、文本、Logo 等）。
- 输出规范化的 JSON 描述，包括每个元素的角色（role）、类型（type）、内容或关键词（description/content）及简单样式提示。
- 示例：从 `"夏日促销海报，需要热带植物背景和醒目的促销标题"` 提取 `background: 热带植物`, `title: "夏季大促销"` 等。

**3. 推荐技能/工具**

1. **自然语言理解引擎**：可调用 Claude 或 GPT 系列大模型，通过对话或指令，让模型按照给定格式返回 JSON。例如构造如下 prompt：“请将以下图像设计需求解析为结构化 JSON，列出画面元素和样式提示。”【51†L330-L339】。优势：大模型可理解模糊描述并生成结构；但需设计好指令以确保输出格式正确。

2. **Graphviz/Diagramming (辅助)**：辅助可视化草图，但难以整合到 Claude Code 中，非首选。

3. **MCP Hook**：如果未来有专门的视觉规划模型/服务，可通过 MCP 形式接入。

**4. 所需权限与外部凭证**

- 网络访问权限（访问 Claude/GPT API 或执行内置 LLM）。
- 不需要文件存储或浏览器权限，除非保存输出 JSON 文件。
- API Key：若使用第三方对话模型服务（OpenAI/Claude API）需相应密钥。
- MCP/插件：未指定。

**5. 错误处理与可观测性**

- **断言输出格式**：检查输出是否为合法 JSON，并包含预期字段（如 `"elements"` 列表）。若解析失败或字段缺失，返回可读错误让模型重试或提示人类介入。
- **回退策略**：若自动解析失败，可回退到逐段提问（分步询问用户更多信息），或采用简单默认方案（如仅提取关键词）。
- **日志/指标**：记录每次解析时的输入、输出和成功率。统计解析时间、失败次数，可用于调优提示和流程。

## 三、Asset Sourcer 设计

**1. 输入/输出格式**

- *输入格式*：来自 Visual Planner 的 JSON，说明了需要哪些素材。例如上节输出的示例 JSON。
- *输出格式*：JSON 列表，包含每个元素的素材来源和URL，例如：

```json
{
  "assets": [
    {
      "role": "background",
      "source": "Unsplash",
      "query": "night sky stars",
      "url": "https://images.unsplash.com/photo-xxxx",
      "license": "Unsplash License"
    },
    {
      "role": "title",
      "source": "generate",
      "prompt": "futuristic style title text",
      "url": "generated_image_id_or_path.png",
      "license": "Generated"
    },
    {
      "role": "logo",
      "source": "internal",
      "path": "/assets/logo.png",
      "license": "Company Asset"
    }
  ]
}
```

*说明*：输出列出每个所需元素的获取方式。`source` 字段可标注使用的API（Pexels/Unsplash/Pixabay）、或标记为“generate”（调用AI生成）、或“internal”（内部素材）。对应的URL或本地路径一并给出。

**2. MVP功能清单**

- 对每个元素（如背景、主体、图标）构建查询关键词或生成提示。
- 依次调用图像API或生成模型，获取合适素材链接。
- 支持多源搜索和降级：若Unsplash无结果，尝试Pexels或Pixabay；若关键词缺失，可使用同义词扩展搜索。
- 下载或缓存素材到本地临时存储。
- 输出上述 JSON 结构，包含所有素材信息。

**3. 推荐技能/工具（优先级排序）**

1. **HTTP API 客户端**（如内置 Fetch/Requests）：用于调用各类图像API（Pexels, Unsplash, Pixabay等）的基础工具。适合精准控制并发与速率。

2. **浏览器自动化（Stagehand/Browser Use/Playwright）**：用于在缺少API的情况下抓取网页。例如直接在浏览器中搜索关键词并下载图片。Stagehand 自称“AI 浏览器自动化框架”【51†L330-L339】；Browser Use 提供了 Claude Code Skill（【13†L440-L448】），可用 `/browser-use run "搜索 X"` 等命令进行搜索。Playwright 则可用于自定义脚本截取屏幕（如保存搜索结果截图【48†L105-L113】），但更适合测试场景。

3. **Pexels API**【20†L59-L68】【20†L139-L148】：免费图库，无版权费用。使用方法：`curl -H "Authorization: API_KEY" "https://api.pexels.com/v1/search?query=..."`【20†L139-L148】。优点：高质量图片、无须付费；缺点：默认速率200次/小时【20†L59-L68】，需在Header中加入API Key。使用例：  
   ```bash
   curl -H "Authorization: YOUR_PEXELS_KEY" \
        "https://api.pexels.com/v1/search?query=night+sky&per_page=5"
   ```
   返回JSON包含图片链接。

4. **Unsplash API**【28†L118-L127】【28†L235-L244】：免费高清图库。使用方法：将 `Client-ID YOUR_ACCESS_KEY` 放在 `Authorization` 头【28†L235-L244】，调用 `GET https://api.unsplash.com/search/photos?query=...`。初始限额为每小时50次【28†L118-L127】。特色：涵盖众多摄影师作品，需在使用时标注来源。优点：大社区贡献；缺点：需申请应用并限流（申请后可提高额度）。示例调用：  
   ```bash
   curl -H "Authorization: Client-ID YOUR_KEY" \
        "https://api.unsplash.com/search/photos?page=1&query=technology"
   ```

5. **Pixabay API**【30†L83-L92】【30†L119-L127】：免版权图库。使用方法：`GET https://pixabay.com/api/?key=YOUR_KEY&q=...`【30†L119-L127】。速率限制：默认每60秒100次请求【30†L83-L92】。图片质量中等，使用前需注册获取Key。优点：无需返回链接版权等（免费授权允许多种用途）；缺点：重复度略高。示例：  
   ```bash
   curl "https://pixabay.com/api/?key=YOUR_KEY&q=forest&image_type=photo&per_page=3"
   ```

6. **OpenAI/Anthropic 图像API**【36†L926-L933】：调用 GPT-Image 或 DALL·E 模型生成或编辑图片。需要付费Key，API格式如：  
   ```bash
   curl -X POST "https://api.openai.com/v1/images/generations" \
        -H "Authorization: Bearer OPENAI_KEY" \
        -H "Content-Type: application/json" \
        -d '{"model":"gpt-image-1","prompt":"夜空下星光闪烁"}'
   ```  
   回应中含生成图片的Base64或URL【36†L926-L933】。优点：高度自定义；缺点：需要API Key，生成图片版权归用户使用许可（注意OpenAI政策）。

7. **Stable Diffusion / Replicate**：可调用如 stability-ai/stable-diffusion 在 Replicate 平台上生成图像。示例（同步调用）：  
   ```bash
   curl -X POST "https://api.replicate.com/v1/predictions" \
        -H "Authorization: Token REPLICATE_TOKEN" \
        -H 'Content-Type: application/json' \
        -d '{
          "version": "模型ID",
          "input": {"prompt": "茂密森林中的阳光"}
        }'
   ```  
  （详见【43†L441-L448】）花费约 $0.0038/次【38†L61-L64】。优点：可免费运行开源Stable Diffusion；缺点：需注册Replicate，生成图片需遵守OpenRAIL许可【38†L87-L95】。

8. **ClipDrop API**【45†L106-L109】：用于自动去背、上色等编辑操作。例如调用去背：  
   ```bash
   curl -X POST https://clipdrop-api.co/remove-background/v1 \
        -H "x-api-key: YOUR_KEY" \
        -F "image_file=@path/to/image.jpg" \
        -o result.png
   ```  
  （见【45†L106-L109】）。消耗限额，默认每分钟60次【45†L93-L99】。优点：方便的背景去除等功能；需申请Key，免费额度有限（100张免费【45†L85-L93】）。

9. **Remove.bg API**【47†L71-L74】：专注去背景，免费前50张/月。示例：  
   ```bash
   curl -H "X-API-Key: YOUR_KEY" \
        -F "image_file=@file.jpg" \
        -f https://api.remove.bg/v1.0/removebg \
        -o no-bg.png
   ```  
   速率和费用见官网，需注意使用条款。

10. **Playwright**【48†L105-L113】：Node/Python自动化浏览器工具。可截屏（`page.screenshot`）和处理DOM。适合需要精确控制网页行为的场景，如要抓取不提供API的图库。优势：高速可靠多浏览器支持【48†L105-L113】；劣势：需要编写脚本。

11. **Stagehand**【51†L330-L339】：结合AI与代码的浏览器自动化框架，可让Claude通过自然语言指令浏览网页【51†L330-L339】。适合复杂多步骤检索，可保持会话。需要API Key、Browserbase账号【51†L399-L403】。

12. **Browser Use**【13†L440-L448】【16†L651-L659】：开源的AI浏览器自动化代理，已为 Claude Code 提供了技能。通过 CLI 或 MCP 与Claude集成，可使用简单命令打开页面、搜索、截图等。优点：即插即用的技能（安装SKILL.md【16†L651-L659】），并发率较高；缺点：需要额外环境（Python）、学习特定命令。

**调用示例（伪代码）**：


**4. 权限与凭证**

- **API Key**：Pexels/Unsplash/Pixabay/Remove.bg/ClipDrop/Replicate/OpenAI 等服务均需注册获取密钥。需将这些Key保密存储，Claude Code 环境需能访问（如读入环境变量）。
- **网络权限**：需要对外互联网访问以调用上述服务。
- **浏览器权限**：若使用 Playwright/Stagehand/Browser Use，需要在环境中安装对应工具，并允许本地或远程浏览器操作。
- **文件存储**：临时存储下载的图片素材（建议写入项目目录或指定缓存路径）。
- **MCP/插件权限**：安装 Browser Use/SKILL.md 技能需要写入 `~/.claude/skills` 目录，MCP可能需要 TCP/Unix 套接字通信权限。

**5. 错误处理与可观测性**

- **请求失败重试**：对各API调用捕获错误（如超时、限流429）。可按服务建议进行指数退避（例如 Remove.bg 文档中提及退避策略【47†L25-L32】）。多次失败后返回错误报告。
- **源冗余策略**：如一API无结果，可自动切换到备用API（Unsplash无图则尝试Pexels）。
- **日志与指标**：记录每次API调用结果、下载的素材数量和大小、失败率、使用的服务类型等。监控API调用配额剩余（如 Pexels 返回头 `X-Ratelimit-Remaining`【20†L180-L183】）。对于浏览器自动化可记录行为步骤和截图以便追踪。
- **断言与可视化检查**：可对下载的图片进行简单断言（如非空、格式正确）。如含图像质量低或尺寸不符，可标记警告。
- **回退措施**：如果所有自动检索失败，可回退到生成图像（如调用生成模型）或使用占位图。

## 四、Image Builder 设计

**1. 工具链与功能模块**

- **图像处理库**：使用 Python Pillow、OpenCV 或 Node.js 的 Sharp、Canvas 等库进行基础操作（裁剪、缩放、滤镜）。它们支持图层合成、透明通道处理。例如，Pillow 可以轻松加载 PNG、合并图层并保存【48†L105-L113】。
  
- **向量图形与模板**：对于需要可伸缩或具有矢量元素的图形，可使用 SVG/Cairo 或 HTML/CSS 模板。一个常见思路是定义 HTML/CSS 版式（包括背景图片、文字位置、样式），然后用无头浏览器（如 Playwright）截屏生成图像，从而自动进行文字换行和自适应布局。也可以使用 LaTeX/PGF 等排版，但中文支持较差。

- **文字排版引擎**：确保文字不会溢出边界。可采用内置换行（Pillow 的 `ImageDraw.multiline_text`）或借助 CSS 浏览器渲染。字体可嵌入项目，需注意中文字体（如 Noto Sans SC）版权。

- **图层管理**：按 Visual Planner 指示排列图层顺序（背景层→主体素材→前景元素→文字）。需要支持透明PNG合成和遮罩。例如，ClipDrop/Remove.bg 结果提供带透明通道图像，可以直接叠加。

- **自动裁剪/对齐**：工具应支持自动裁剪图片（如按照主要对象区域）和对齐文字。可使用人脸或物体检测（OpenCV Haar/深度学习）实现自动裁剪，也可简单地按设计稿指定裁剪区域。

- **多分辨率导出**：输出不同尺寸（标准和高清）。方案：先生成高分辨率（如 2K），然后缩放。或采用向量/HTML生成后导出不同dpi。

- **示例工作流（Mermaid 流程图）**：

```mermaid
flowchart TD
  A[Visual Planner 输出的设计 JSON] --> B[图像合成准备]
  B --> C[下载/载入素材文件]
  C --> D[自动裁剪/调整尺寸]
  D --> E[文字内容填充]
  E --> F[图层合成 (背景/主体/文字等)]
  F --> G[后处理 (滤镜、水印、去雾等)]
  G --> H[导出最终图像]
  H --> I[生成多分辨率版本 (标准/Retina)]
```

**2. 关键实现点**

- **自动裁剪**：对于背景或主体照片，可按需要裁剪中心区域或使用简单算法（如侦测主要物体边界）自动裁剪。参考Playwright截图【48†L105-L113】的 `fullPage: true` 功能，可保存完整页面图像。

- **文字排版**：根据布局提示居中或对齐。使用等宽或无衬线字体确保可读性。处理长文本时自动换行或缩放字体直至适配预设区域。可在 HTML/CSS 中使用 `text-wrap` 或 CSS Grid 简化。

- **图层管理**：在代码中按顺序叠加图像图层。比如 Pillow 的 `alpha_composite` 能正确处理透明PNG。确保背景最先渲染，后续图层覆盖前层上不透明部分。

- **模板系统**：维护一套可复用模板（如预定义多种海报风格的 JSON/CSS 模板），代码按照模板变量动态填充（如标题文本、配色）。

- **导出**：输出格式可包括 PNG、JPEG 等。多分辨率示例：若目标为网页显示，可输出 1x 和 2x。对于矢量图（如生成SVG），可直接导出任意大小。

**3. 示例工作流详解**

1. **准备素材**：根据 Asset Sourcer 输出的URL列表下载所有图像素材到临时目录。
2. **裁剪/调整**：对超过画布尺寸的图片执行缩放或裁剪；对人物或物体图片可尝试自动裁剪中心对象。
3. **合成排版**：创建一个画布（例如 1920x1080），按布局将背景图拉伸铺满，依次粘贴其他图层。将文本渲染到指定位置，注意字体、颜色。
4. **后期处理**：可加统一滤镜（如色调），或叠加公司的水印。确保输出风格统一。
5. **生成变体**：在输出高清图后，用图像库生成缩小图（Retina版减半分辨率），确保各尺寸对应不同使用场景。

**4. 所需权限**

- **文件访问**：写入和读取下载的图片，以及最终合成图像文件。
- **字体资源**：可能需要加载字体文件，确保可嵌入输出（如 .ttf 存放于工程文件夹）。
- **运行时权限**：如果使用无头浏览器截图（如Playwright），需启动浏览器进程。

**5. 错误处理与可观测性**

- **尺寸/格式检查**：合成前检查图像尺寸是否满足要求。如素材过小导致模糊，可记录警告或请求重试别的素材。
- **断言布局**：确保所有文本和重要元素未越界。可在合成后计算文本边界，若超出则尝试减小字体或重新布局，并记录成功/失败。
- **日志与指标**：记录每张合成图的输出尺寸、处理时间、使用的滤镜/特效。监控失败合成次数和质量问题反馈（例如使用视觉对比检查）。
- **回退方案**：若某素材加载失败或不符合规格，可使用占位颜色或默认图案替代，继续合成；如文字超长，可截断并加省略号。

## 六、示例执行指令

以下步骤以独立指令形式给出，可供 Claude Code 逐步执行与验证。每步包含目标、输入示例、预期输出、调用技能/API示例和错误回退方案。

1. **步骤 1: Visual Planner 解析**
   - **目标**：将用户图像需求文本解析为结构化 JSON 设计方案。
   - **输入示例**：`"生成一个庆典主题海报，背景是热气球，标题文字‘欢庆时刻’"`
   - **期望输出**（JSON 示例）：
     ```json
     {
       "elements": [
         {"role":"background","type":"image","description":"热气球升空，颜色鲜艳"},
         {"role":"title","type":"text","content":"欢庆时刻","font":{"size":64,"color":"白色","style":"无衬线加粗"},"position":"center"},
         {"role":"foreground","type":"image","description":"欢庆焰火或彩带"}
       ],
       "theme":"庆典,喜庆"
     }
     ```
   - **调用技能/API**：使用 Claude 对话模型（如 ChatCompletion）处理。示例调用：  
     ```
     Prompt: "请将以下图像设计需求解析为 JSON 规划：'生成一个庆典主题海报...'。输出包括 elements 列表等。"
     ```
   - **错误回退**：若输出非JSON格式或缺失字段，提示模型修正格式；或拆分成多次提问（先提问背景元素，再提问文本元素）。

2. **步骤 2: 关键词提取与准备**
   - **目标**：从 Visual Planner 输出的 JSON 提取搜索关键词和生成提示。
   - **输入示例**：来自步骤1的 JSON，如 `{"elements":[{"role":"background","description":"热气球..."}]}`。
   - **期望输出**（JSON）：  
     ```json
     {
       "background_query": "hot air balloon colorful celebration",
       "foreground_query": "festival fireworks",
       "title_text": "欢庆时刻",
       "title_style": "white bold sans-serif"
     }
     ```
   - **调用技能/API**：简单解析或再次调用 LLM。如通过代码提取字段。示例：`title_text = elements[1]["content"]` 等。
   - **错误回退**：若JSON解析失败，提示重新确认Visual Planner输出；或使用默认关键字。

3. **步骤 3: 素材检索（图库API）**
   - **目标**：使用合适的图库API获取“热气球背景”图片。
   - **输入示例**：`{"query":"hot air balloon colorful celebration","role":"background","source":"Pexels"}`
   - **期望输出**：JSON 包含图片URL，例：  
     ```json
     {"role":"background","url":"https://images.pexels.com/photos/2344...","license":"Pexels"}
     ```
   - **调用技能/API**：HTTP请求工具调用 Pexels API 示例：  
     ```
     GET https://api.pexels.com/v1/search?query=hot+air+balloon+celebration&per_page=1
     Header: Authorization: YOUR_PEXELS_KEY
     ```
   - **错误回退**：如果返回空结果，改用 Unsplash API：  
     ```
     GET https://api.unsplash.com/search/photos?query=hot+air+balloon...
     ```
     并记录使用的服务。若两者都失败，改为直接调用浏览器技能检索或使用生成模型。

4. **步骤 4: 素材下载与缓存**
   - **目标**：下载步骤3获得的图片到本地。
   - **输入示例**：`{"role":"background","url":"...pexels...jpg"}`
   - **期望输出**：本地文件路径，如 `"/cache/bg1.jpg"`，并记录成功标志。
   - **调用技能/API**：可使用内置文件下载或 Node/Python 下载库。例如 Python:  
     ```python
     import requests
     r = requests.get(url)
     open('/cache/bg1.jpg','wb').write(r.content)
     ```
   - **错误回退**：如果下载失败（如网络断开、403），重试或换用镜像URL；记录失败日志以便重查。

5. **步骤 5: 图像合成准备**
   - **目标**：对所有下载素材进行必要处理（裁剪/缩放）。
   - **输入示例**：背景图 `"/cache/bg1.jpg"`、标题文本等。
   - **期望输出**：处理后素材路径或图像对象，例 `"/cache/bg1_resized.jpg"`。
   - **调用技能/API**：使用图像库（如 Pillow）处理。示例：  
     ```python
     from PIL import Image
     img = Image.open("/cache/bg1.jpg")
     img = img.resize((1920,1080), Image.ANTIALIAS)
     img.save("/cache/bg1_resized.jpg")
     ```
   - **错误回退**：如果图像尺寸极小，尝试不裁剪直接合成，或者标记该素材质量低；若库报错则尝试使用 ImageMagick CLI。

6. **步骤 6: 图层合成与排版**
   - **目标**：在画布上按照方案叠加背景、其他素材并绘制文本。
   - **输入示例**：裁剪后的背景图、标题文本“欢庆时刻”、字体样式等。
   - **期望输出**：最终合成图像文件（如 `/output/final.png`）。
   - **调用技能/API**：Pillow 或无头浏览器。例如 Pillow 代码示例：  
     ```python
     canvas = Image.new("RGB",(1920,1080),(255,255,255))
     bg = Image.open("/cache/bg1_resized.jpg")
     canvas.paste(bg,(0,0))
     draw = ImageDraw.Draw(canvas)
     font = ImageFont.truetype("NotoSansSC-Bold.otf", 64)
     draw.text((960,50),"欢庆时刻",font=font,fill=(255,255,255),anchor="ms")
     canvas.save("/output/final.png")
     ```
   - **错误回退**：如果文字超出边界，可尝试减少字体大小或换行；如缺少字体文件，使用系统默认或其它字体。

7. **步骤 7: 多分辨率导出**
   - **目标**：生成低分辨率缩略图或 Retina 版本。
   - **输入示例**：高分辨率图 `/output/final.png`。
   - **期望输出**：例如 `/output/final@2x.png`, `/output/final_thumb.png`。
   - **调用技能/API**：再次使用图像库，调整尺寸保存。示例：  
     ```python
     hi = Image.open("/output/final.png")
     hi.save("/output/final@2x.png")  # 假设原图已为高清，可直接复制
     lo = hi.resize((hi.width//2, hi.height//2), Image.ANTIALIAS)
     lo.save("/output/final_thumb.png")
     ```
   - **错误回退**：如因内存不足无法处理超大图，可逐步缩放或使用磁盘缓存；记录输出完成时间。

每步完成后建议检查输出格式与内容是否符合预期，并在必要时返回错误信息给Claude Code，以决定重试或调整。这样分步执行有助于逐项验证并快速定位问题。
