# CLAUDE.md

此文件提供 Claude Code 在本仓库中的全局工作规则。

## 仓库目标

本仓库用于在 Obsidian 中管理中文图文自媒体内容生产流程，账号方向为：

**前沿 AI 信息研究与应用分享**

当前默认优先支持**小红书图文内容**，并为**今日头条**、**微信公众号**预留适配能力。

## 默认工作流

Claude 在本仓库中的默认流程：

1. 先判断选题是否值得做
2. 再整理资料与对标信息
3. 再核验关键信息与事实表述
4. 先生成平台无关母稿
5. 再适配为不同平台版本
6. 最后审校并整理为待发布内容

未经明确指令，不执行发布动作。

## 内容状态流转

选题池 → 待写 → 写作中 → 审稿中 → 待发布 → 已发布 → 复盘

目录流转：

- `03_选题与素材/选题池/`：选题沉淀
- `02_内容项目/待写/`：准备写作
- `02_内容项目/审稿中/`：完成初稿待审核
- `02_内容项目/待发布/`：审核通过待发布
- `02_内容项目/已发布/`：已发布内容
- `05_发布与复盘/`：数据记录与复盘分析

## 目录职责

- `00_导航/`：入口、索引、看板，不存放正文
- `01_定位与规则/`：账号定位、平台规则、写作规范
- `02_内容项目/`：内容生产主战场
- `03_选题与素材/`：选题、灵感、素材、对标拆解
- `04_知识卡片/`：可复用观点、表达、结构沉淀
- `05_发布与复盘/`：发布记录、数据记录、复盘
- `90_模板/`：模板文件库
- `99_附件/`：图片、截图、封面等附件

## 协作原则

信息源优先级：

`CLAUDE.md` → `01_定位与规则/` → `90_模板/` → 当前项目文件

工作要求：

- 先读规则，再生成或修改内容
- 优先基于模板产出，不随意从零发挥
- 默认先写母稿，再做平台适配
- 默认优先适配小红书
- 修改已有内容时，优先保持原有结构与意图
- 不臆造案例、数据、平台机制
- 不写明显 AI 腔、套话、口号式表达

## 禁止事项

- 跳过仓库规则直接生成内容
- 将长篇正文写入导航类文件
- 随意改变目录结构、命名体系
- 无依据地补充事实、案例、数据
- 未经核验写“最新”“重磅”“已确定”等表述
- 擅自执行发布、提交、同步、覆盖等有副作用动作

## 扩展约定

如后续引入 `.claude/agents/`、`.claude/rules/`、`.claude/skills/`：

- 根 `CLAUDE.md` 只负责全局目标、默认流程和硬性约束
- `rules` 负责平台细则与写作规范
- `agents` 负责专门角色
- `skills` 负责可重复、可触发的固定动作

## 虚拟环境与脚本执行

- 根目录使用 `uv` 管理 Python 虚拟环境
- `.venv/` 为 uv 创建的虚拟环境目录
- Python 脚本可调用 `.claude/skills/` 中的 Node.js 技能
- 通过子进程执行外部技能时需指定正确的工作目录
